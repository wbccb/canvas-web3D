结论：

1. 我们ctx.drawImage(0,0)时会将 整个image绘制到canvas.width+canvas.height上面

1.1 image的大小>canvas.width/height，则会被截断，除非我们drawImage(0,0, img.width, img.height, 0, 0, canvas.width, canvas.height)进行缩放

1.2 image的大小 < canvas.width/height，则无法充满

2. 在涉及到scale和canvas.width=canvas.style.width*dpr的情况时，我们只要考虑一开始image是否>canvas.width/height

2.1 比如canvas.width=80;image.width=96；canvas.style.width=120；因为一开始绘制时image.width>canvas.width，因此会被截断，就算后续canvas.width为了适配canvas.style.width进行自适应放大，放大的也是那被截断的部分的放大；后续用了scale，也是对被截断的部分的缩小，也就是下面id="4"那一部分的逻辑

2.2 而id="6"那一份的逻辑，则是canvas.width刚刚好等于image.width，那么就能正常显示图片，但是由于canvas.width/canvas.style.width=2，因此得缩小一半体积，此时的图片也是完美展示的！但是好巧不巧，还要ctx.scale(2,2)，因此此时图片再次放大到96+96，此时的容器只有48+48，图片溢出显示只有4分之1！（可以注释id="6"的ctx.scale()代码进行验证！）

2.3 那如果是canvas.width>image.width，并且刚刚好canvas.width=image.width*dpr呢？由上面的分析可以知道，会无法充满，然后由于canvas.width/canvas.style.width=dpr，因此整体会缩放dpr倍，此时image也会缩放dpr倍，然后就是ctx.scale(dpr, dpr)，会让image再放大dpr倍，因此image的大小等于没变化，而此时的image其实就等于canvas.style.width，因此我们是根据image.width去设置canvas.style.width,因此此时image刚刚好充满整个容器
